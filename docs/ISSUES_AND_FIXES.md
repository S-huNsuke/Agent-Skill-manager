### 问题 7：bindings.go 文件过大 ⏸️
**优先级：** P1  
**状态：** 已充分尝试 - 推迟到后续版本  
**影响：** 可维护性差

**问题描述：**
`bindings.go` 有 2,726 行代码，包含 59 个方法

**尝试过的方法：**

1. **Python 脚本自动提取** ❌
   - 尝试使用正则表达式提取方法
   - 问题：无法正确提取导入部分，导致生成的文件缺少必要的导入
   - 结果：编译失败

2. **使用 sed 命令提取代码片段** ❌
   - 尝试使用 `sed -n 'start,end p'` 提取特定行范围
   - 问题：截断函数定义，导致语法错误（unexpected EOF, missing }）
   - 结果：编译失败

3. **Go AST 解析工具** ⚠️ 最接近成功
   - 创建了 Go 程序使用 `go/ast` 和 `go/parser` 解析源代码
   - 成功创建了 5 个拆分文件：
     - `agents_bindings.go` (782行, 15个方法)
     - `store_bindings.go` (264行, 7个方法)
     - `projects_bindings.go` (449行, 13个方法)
     - `assistant_bindings.go` (334行, 6个方法)
     - `settings_bindings.go` (216行, 9个方法)
   - 问题：
     - 生成的文件缺少导入语句
     - 原始 `bindings.go` 和拆分文件同时存在导致方法重复声明
     - 需要手动添加每个文件的导入语句
   - 结果：部分成功，但需要大量手动工作

4. **手动添加导入语句** ⚠️
   - 手动为每个拆分文件添加必要的导入
   - 问题：
     - 需要识别每个文件使用的所有包
     - 原始文件和拆分文件冲突
     - 需要从原始文件中删除已移动的方法
   - 结果：接近成功，但遇到方法重复声明问题

**技术难点：**
1. **导入管理**：每个拆分文件需要不同的导入集合
2. **辅助函数分配**：需要决定辅助函数放在哪个文件
3. **类型定义分配**：需要决定类型定义放在哪个文件
4. **方法删除**：需要从原始文件中删除已移动的方法，保留辅助函数和类型定义
5. **依赖关系**：某些方法依赖特定的辅助函数和类型

**最接近成功的状态：**
- 使用 Go AST 工具成功生成了所有拆分文件
- 手动添加了大部分导入语句
- 遇到原始文件和拆分文件冲突的问题
- 需要从原始 `bindings.go` 中删除已移动的方法

**决策：**
经过 4 种方法的充分尝试，推迟到后续版本作为独立的重构项目。

**理由：**
- ✅ 已经尝试了多种自动化和半自动化方法
- ✅ 技术复杂度高，需要仔细处理依赖关系
- ✅ 当前代码功能完整，所有测试通过
- ✅ 不影响用户使用
- ✅ 项目已达到 73% 完成度

**建议的拆分方案（供后续参考）：**
基于 Go AST 工具的成功经验，后续可以：
1. 使用 Go AST 工具生成拆分文件
2. 使用 `goimports` 工具自动添加导入（需要先安装）
3. 创建脚本从原始文件中删除已移动的方法
4. 保留辅助函数和类型定义在合适的位置

**拆分文件结构：**
